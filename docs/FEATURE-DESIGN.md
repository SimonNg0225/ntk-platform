# NTK Platform · 功能設計（深入版）

> 針對 6 個優先功能嘅詳細設計：資料模型、畫面、互動、互相連動。
> 學習模式：**知識卡 + 間隔複習**、**學習儀表板**
> 工作模式：**課程進度追蹤**、**教學資源庫**、**成績管理**、**BAFS 題庫**

---

## 0. 架構洞見：兩條共用骨幹

工作模式 4 個功能全部圍住 **BAFS 課題** + **班別/學生** 兩個共用資料層轉。
**建議先起呢兩層**，其餘功能砌上去就乾淨又互通。

```
            BAFS 課題 (curriculum_topics)
   ┌───────────┬──────────┬───────────┐
 課程進度     資源庫      題庫        成績
   └───────────┴── 班別/學生 ─────────┘
```

學習模式：`知識卡` → 餵 `儀表板`；加 `活動記錄` → 撐 streak。

## 1. 📘 知識卡 + 間隔複習 (Flashcards + SRS)

### 概念
牌組 (deck) 入面有卡 (card，正面/背面)。系統用**間隔重複**安排幾時複習：
記得就拉長下次複習嘅間隔，唔記得就好快再出，務求用最少時間記得最牢。

### 資料模型
| 表 | 欄位 |
| --- | --- |
| `decks` | id, user_id, name, description, created_at |
| `cards` | id, deck_id, user_id, front, back, **ease**(預設2.5), **interval_days**(0), **repetitions**(0), **due_date**, last_reviewed, created_at |

### SRS 演算法（SM-2 精簡版）
複習時用家自評 4 個程度，更新該卡：
| 評分 | 處理 |
| --- | --- |
| 😵 唔記得 (Again) | repetitions=0；interval=0（今日內再出）；ease −0.2（最低 1.3） |
| 😓 有啲難 (Hard) | interval = interval × 1.2；ease −0.15 |
| 🙂 記得 (Good) | 第1次=1日、第2次=6日、之後 interval × ease |
| 😎 好易 (Easy) | interval × ease × 1.3；ease +0.15 |
更新後 `due_date = 今日 + interval_days`。

### 畫面
1. **牌組列表**：每個牌組顯示「新卡 / 今日到期 / 總數」。
2. **牌組內容**：卡列表、新增 / 編輯 / 刪除卡。
3. **複習模式**：逐張出（淨係出到期嘅）→ 撳「翻面」睇答案 → 揀 4 個評分 → 自動排下一張；完成顯示今日成績。

### 連動
- 儀表板顯示「今日要複習 N 張」。
- 未來 AI：由筆記 / 主題自動生成一疊卡。

## 2. 📘 學習儀表板 (Learning Dashboard)

### 概念
學習模式嘅「今日總覽」首頁，一眼睇晒最重要嘢 + 快速動作。

### 小工具 (widgets)
- **今日複習**：到期卡數 → 撳入即開複習。
- **目標進度**：平均完成度 + 快到期目標。
- **連續學習日數 (streak)**：連續有活動嘅日數 🔥。
- **本週統計**：複習咗幾多張 / 專注時間（如有計時器）。
- **最近筆記**：近 3 條。
- **快速動作**：＋筆記、開始複習。

### 資料
大部分由現有表計出；新增 `activity_log`(date, type, count) 支撐 streak / 本週統計。

## 3. 💼 課程進度追蹤 (Curriculum Progress)

### 概念
對住 BAFS 課程大綱，記低**每班**教到邊個課題、進度幾多。

### 資料模型
| 表 | 欄位 |
| --- | --- |
| `curriculum_topics` | id, user_id, **part**(必修/選修), area(課題範疇), topic(課題), order |
| `class_progress` | id, user_id, class_id, topic_id, **status**(未開始/進行中/完成), date_done, notes |

### 畫面
- **矩陣檢視**：列＝課題、欄＝班別、格仔用顏色顯示狀態 ✅🟡⬜。
- **單班檢視**：該班所有課題 checklist + 完成百分比。

### 連動
- 課題連住 **資源庫**（嗰個課題有咩教材）同 **題庫**（嗰個課題有咩題）。
- 成績差嘅課題 → 提示重教。

### BAFS 課題（建議預載，可自行修改）
**必修部分**
- 商業環境：香港營商環境、企業擁有權形式、商業道德與社會責任
- 管理入門：管理職能（計劃/組織/領導/控制）、商業主要職能
- 會計入門：會計目的與角色、會計原則、複式記賬、財務報表
- 個人理財基礎：個人理財計劃、風險管理與保險、貨幣時間價值

**選修（二揀一）**
- *會計單元*：原始分錄與分類賬、銀行往來調節、管制賬戶、錯誤更正與調整、各類企業財務報表、不完整記錄、會計比率分析、成本分類、成本計算法（分批/邊際/吸納）、本量利分析、預算、標準成本
- *商業管理單元*：財務管理、人力資源管理、市場營銷管理、生產營運管理

## 4. 💼 教學資源庫 (Teaching Resource Library)

### 概念
集中存放 / 整理教材，按課題、類型、標籤搵到。

### 資料模型
| 表 | 欄位 |
| --- | --- |
| `resources` | id, user_id, title, **type**(講義/PPT/試題/連結/影片/筆記), url 或 file_path, **topic_id**, class_ids[], tags[], notes, created_at |

### 畫面
- **資源列表**（卡或表）+ 篩選：按課題 / 類型 / 標籤 / 搜尋。
- **新增**：標題、類型、連結或上傳檔、課題、標籤。
- **詳情**：開連結 / 預覽、筆記。

### 儲存
- 連結類即刻得；檔案類用 **Supabase Storage**（之後階段）。

### 連動
課程進度嘅課題 → 一撳睇晒嗰課題嘅資源；備課時直接拉。

## 5. 💼 成績管理 (Gradebook)

### 概念
記錄學生各評估分數，自動計平均、睇邊個 / 邊個課題弱。

### 資料模型
| 表 | 欄位 |
| --- | --- |
| `students` | id, user_id, class_id, name, student_no |
| `assessments` | id, user_id, class_id, name, **type**(測驗/考試/功課), date, max_score, weight, **topic_id** |
| `scores` | id, assessment_id, student_id, score, remark |

### 畫面
- **單班成績表**：學生 × 評估 嘅表格，直接入分。
- **自動計算**：每生平均、每評估平均、班平均、加權總分。
- **洞察**：低於門檻嘅學生、最弱課題（評估連住 topic_id）。

### 連動
- 弱課題 → 回饋去課程進度（建議重教）+ 題庫（出多啲練習）。
- 未來 AI：根據分數草擬評語。

---

## 6. 💼 BAFS 題庫 (Question Bank)

### 概念
按課題 / 難度 / 題型儲題目，可組卷；之後接 AI 自動出題入庫。

### 資料模型
| 表 | 欄位 |
| --- | --- |
| `questions` | id, user_id, **topic_id**, **type**(MC/短答/長題/個案), **difficulty**(易/中/難), stem(題目), options[](MC用), answer, marks, source, tags[] |
| `papers`(之後) | id, name, question_ids[] |

### 畫面
- **題目列表** + 篩選（課題 / 題型 / 難度）+ 搜尋。
- **新增 / 編輯題目**。
- **組卷**：揀題 → 預覽 / 匯出 / 列印。

### 連動
- topic_id 連課程；AI 出題直接入呢個庫；成績嘅評估可連返題目。

---

## 7. 建議落地次序

1. **共用骨幹**：`curriculum_topics`（預載 BAFS）+ `班別/學生`
2. **課程進度追蹤**（即刻睇到骨幹價值）
3. **題庫** + **資源庫**（掛上課題）
4. **成績管理**（掛上班別/學生 + 課題）
5. 學習模式：**知識卡 + SRS** → **學習儀表板**

> 注意：以上資料模型同 Supabase 表一致，所以而家可以先用本機 (localStorage) 整原型，之後接 Supabase 直接對得上，唔使重寫。
